C Primer Plus Sixth Edition - 第三章 数据和C 总结

2018.06.13 一白 C  热度 ℃

数据类型

基本数据类型 : 由11个关键字组成：int , long , short , char, float, double, signed , unsigned , _Bool , _Complex , _Imaginary 。

在不同的系统上，这些类型占据的字节长度不同，C标准对基本数据类型只规定了允许的最小大小，其中 short 占用的存储空间不能多于 int , long 占用的存储空间不能少于 int。

目前个人计算机上最常见的设置是:

  • short / short int : 2 byte
    // 最大的short类型整数 <= 最大的int类型整数。 C语言规定至少占16位;
  • int : 2 或 4 byte
    // 依计算机的自然字长而定。C语言规定不小于16位;
  • long / long int : 4 byte
    // 该类型可表示的整数 >= 最大的int类型整数。C语言规定至少占32位;
  • long long / long long int : 8 byte
    // 该类型可表示的整数 >= 最大的long类型整数。C语言规定至少占64位;
  • float : 4 byte;
  • double : 8 byte;
  • char : 1 byte;

原则上，short, int, long, long long这4种类型代表4种不同大小，但在实际使用中，有些类型之间通常有重叠。
具体可用 sizeof() 测试一下。

可移植类型：stdint.h和 inttypes.h

  • 精确宽度类型：eg: int32_t, 32位有符号整数类型，整数类型的宽度正好是32位
  • 最小宽度类型: eg: int_least8_t, 可容纳8位有符号整数值的类型中宽度最小的类型的一个别名
  • 最快最小宽度类型: eg: int_fast8_t, 系统中对8位有符号值而言运算最快的整数类型的别名

数制

数制 表示方法 显示各进制数制的前缀
八进制 前缀“0”, %o %#o
十进制 无前缀 , %d
十六进制 前缀“0x” , %x %#x

标记

标志 意义
# 显示各进制数制的前缀,见上表格事例
- 输出数据向左对齐，数据向左靠
+ 有符号的值若为正，则显示带加号的符号；若为负，则显示带减号的符号
（空格） 有符号的值若为正，则显示时带前导空格(但是不显示符号);若为负，则带减号符号。+标志会覆盖空格标志
0 对于所有的数字格式，用前导零而不是空格填充字段宽度。如果出现-标志或者指定了精度(对于整数)则忽略该标志


  

格式字符转换

转换说明 数据类型 备注
%d,%i int 输出有符号的十进制整数
%ld long int l,ll和整数转换符一起使用，类似：%lo, %lx, %lu; %lld (long long int)
%hd short int，unsigned short int h和整数转换符一起使用，类似：%ho, %lx
%u unsigned int 无浮点型整数
%c char
%f float,double
%Lf long double 和浮点数转换说明符一起使用，类似：%Lf, %10.4Le
%a,%A 浮点数、十六进制数和p计数法(C99/C11)
%e,%E 浮点数，以指数形式输出 2.87e-3, .2e5
%g,%G 根据数值不同自动选择%f或%e,%e格式在指数小于-4或者大于等于精度时使用
%s 输出一个字符串 c语言中无字符串类型，要添加头文件<string.h>
%o 八进制输出int or long int or short int 显示八进制数制前缀:%#o
%x,%X 以16进制输出 %#x
%zd sizeof返回类型, 如要打印sizeof(int)的值 一些不支持C99和C11的编译器可用%u或%lu代替%zd，类似：%zd，%12zx
%p 指针
%% 打印一个百分数

f格式符：

  1. %f: 系统处理一般方法是整数部分全部输出，小数部分输出6位
  2. %m.n: 指定数据宽度和小数位数。总的字符长度m位，小数点后n位
  3. %-m.n: 输出数据向左对齐，数据向左靠，长度不超过m时右端补空格
